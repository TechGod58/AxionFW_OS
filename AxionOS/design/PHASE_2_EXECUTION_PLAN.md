# AxionOS Phase 2 Execution Plan

## Objective
Move from design-heavy scaffolds to runnable shell/app components with VM-first enforcement.

## Track A — Shell Runtime
1. Context Menu Host (full menu, no "show more options")
2. Taskbar host implementation
3. Start menu host implementation
4. Settings shell navigation host

## Track B — Core Apps (MVP implementations)
1. Clock
2. Calendar
3. Pad
4. Pulse Monitor
5. Capture

## Track C — Security/Policy Integration
1. Enforce APP_VM_ENFORCEMENT at launch points
2. Wire promotion gate for app persistence paths
3. Hook corr trace into shell actions

## Track D — Packaging
1. Build repeatable release generator
2. Hash/sign manifest pipeline
3. Installer media creator scaffold

## Exit Criteria for Phase 2
- User can launch core shell + 3 core apps through capsule policy
- Right-click full menu works without fallback submenu
- Promotion/audit traces visible for app write actions
- Release output generated by one command
