# AXION Apps Category Spec v1

Purpose: close out Apps category for install/defaults/startup/background governance.

## Sections
- Installed apps list
- Default apps mapping
- Startup/background permissions
- Optional components/add-ons
- App repair/reset/uninstall actions

## Actions
- Set default app per file/protocol class
- Enable/disable startup permission
- Enable/disable background permission
- Request repair/reset/uninstall
- Open app storage usage detail

## Done criteria
- Runtime host persists app settings/state
- Corr-traced events for all app control actions
- Integrates with startup/background policy + VM launch policy
